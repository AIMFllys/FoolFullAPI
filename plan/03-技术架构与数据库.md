## 前后端通信协议

所有 API 请求使用 JSON 格式，响应统一结构：

```typescript
// 成功响应
interface SuccessResponse<T> {
  success: true;
  data: T;
}

// 错误响应
interface ErrorResponse {
  success: false;
  error: {
    code: string;       // 如 "AUTH_FAILED", "BANNED", "RATE_LIMIT"
    message: string;    // 用户友好的错误消息
  };
}
```

## 数据库设计

### 选型理由

- 无需额外安装数据库服务器
- 单文件存储，便于备份和清理（4/2 直接删文件）
- 短期活动（1~2 天），性能完全足够
- WAL 模式优化并发

### users 表

```sql
CREATE TABLE IF NOT EXISTS users (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  username      TEXT    NOT NULL UNIQUE,
  password      TEXT    NOT NULL,              -- bcrypt 哈希
  api_key       TEXT    UNIQUE,                -- UUID v4
  step          INTEGER NOT NULL DEFAULT 1,    -- 当前步骤 (1~6)
  ask_count     INTEGER NOT NULL DEFAULT 0,    -- 已使用交互次数
  is_banned     INTEGER NOT NULL DEFAULT 0,    -- 是否封禁 (0/1)
  is_ide_mode   INTEGER NOT NULL DEFAULT 0,    -- 是否 IDE 模式 (0/1)
  first_message TEXT,                          -- 用户第一条消息（Step 2~5 需要引用）
  created_at    TEXT    NOT NULL DEFAULT (datetime('now')),
  updated_at    TEXT    NOT NULL DEFAULT (datetime('now'))
);
```

> `first_message` 字段：存储用户在当前循环中发送的第一条消息。Step 2~5 需要引用这条消息来生成个性化回复。每次 step 重置为 1 时清空。

### chat_logs 表（可选，用于统计）

```sql
CREATE TABLE IF NOT EXISTS chat_logs (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id     INTEGER NOT NULL,
  step        INTEGER NOT NULL,
  user_msg    TEXT    NOT NULL,
  ai_reply    TEXT    NOT NULL,
  model_used  TEXT    NOT NULL DEFAULT 'none', -- 'kimi' / 'deepseek' / 'template' / 'hardcode'
  source      TEXT    NOT NULL DEFAULT 'web',  -- 'web' 或 'api'
  created_at  TEXT    NOT NULL DEFAULT (datetime('now')),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### global_stats 表

```sql
CREATE TABLE IF NOT EXISTS global_stats (
  id              INTEGER PRIMARY KEY DEFAULT 1,
  total_users     INTEGER NOT NULL DEFAULT 0,
  total_asks      INTEGER NOT NULL DEFAULT 0,
  daily_api_count INTEGER NOT NULL DEFAULT 0,
  last_reset_date TEXT    NOT NULL DEFAULT (date('now'))
);
```

### 索引

```sql
CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_api_key ON users(api_key);
CREATE INDEX idx_chat_logs_user_id ON chat_logs(user_id);
```

### 初始化

```sql
INSERT OR IGNORE INTO global_stats (id) VALUES (1);
```
