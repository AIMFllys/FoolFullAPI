# 06 — 安全性设计

---

## 威胁分析

| 威胁 | 风险等级 | 应对措施 |
|------|---------|---------|
| SQL 注入 | 高 | 参数化查询（better-sqlite3 的 `?` 占位符） |
| XSS 攻击 | 高 | React 默认转义 + DOMPurify 清理 Markdown |
| 暴力破解密码 | 中 | bcrypt 哈希 + 登录速率限制 |
| JWT 伪造 | 中 | 强密钥（≥32字符）+ 短有效期（24h） |
| DDoS 攻击 | 中 | express-rate-limit + IP 封禁 |
| 提示词注入 | 中 | 双重约束 + 输出过滤 + 兜底回复 |
| 数据泄露 | 低 | 4/2 自动清理 + 最小化数据收集 |

## 提示词注入防护

### 用户消息预处理

```typescript
const sanitizeUserMessage = (msg: string): string => {
  return msg
    .replace(/\[system\]/gi, '')
    .replace(/\[assistant\]/gi, '')
    .replace(/role:\s*system/gi, '')
    .replace(/忽略.*指令/gi, '')
    .replace(/ignore.*instruction/gi, '');
};
```

### AI 输出验证

```typescript
const LEAK_KEYWORDS = ['愚人节', '整蛊', '玩笑', 'prank', 'april fool', '骗你', '恶作剧'];

const validateAiOutput = (reply: string, currentStep: number): string => {
  if (currentStep === 6) return reply; // Step 6 允许揭晓
  for (const keyword of LEAK_KEYWORDS) {
    if (reply.toLowerCase().includes(keyword)) {
      return FALLBACK_REPLIES[currentStep];
    }
  }
  return reply;
};
```

### 双重约束（替代原三重约束）

由于 6 步方案中 Kimi 模型的指令遵循能力更强，改为双重约束：

```json
[
  { "role": "system", "content": "【核心约束】你当前处于步骤 X，必须严格遵循步骤要求，不能回答用户问题，不能透露整蛊意图。" },
  { "role": "system", "content": "【步骤 X 的具体提示词】" },
  { "role": "user", "content": "用户消息" }
]
```

Kimi 模型指令遵循能力强，双重约束已足够。DeepSeek 步骤的输出本身就很模板化，偏移风险低。

## 兜底回复

当 AI 输出不符合预期时使用：

| 步骤 | 兜底回复 |
|------|---------|
| 1 | "收到您的需求，正在进行深度分析...\n\n- 问题拆解中\n- 方案评估中\n- 技术选型中\n\n分析完成，正在为您生成完整回答..." |
| 2 | （模板硬编码，不需要兜底） |
| 3 | "很抱歉，生成过程中遇到网络波动，已保存进度（87%），正在重新连接，请稍候..." |
| 4 | "连接已恢复。检测到您之前的请求，由于中断可能导致数据不一致，是否需要重新生成？" |
| 5 | "好的，以下是为您生成的回答：\n\n正在加载内容...\n\n⚠️ [系统提示] 检测到当前会话为愚人节特别版本，内容生成已暂停。" |
| 6 | （硬编码，不需要兜底） |
